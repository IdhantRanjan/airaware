# Air Quality Monitoring Platform — Presentation Writeup

## What We Built (Overview)

A real-time air-quality monitoring dashboard powered by real sensor data and machine learning. Data comes from [DrKoz/ACS_AQ](https://github.com/DrKoz/ACS_AQ) — 5 PurpleAir-style sensors, Nov 1–29, 2025, 2-minute resolution. We run a Python ML pipeline (trust scores, event detection, forecasting), export results to JSON, and the React dashboard fetches and displays them.

## Machine Learning — What We Did (Detailed)

### 1. Trust Score

**Purpose:** Quantify how much to trust each reading using dual-sensor agreement.

**Formula:** `trust = 100 · e^(-3 · |A−B|/x̄)` where A = `corr_pm2.5_a`, B = `corr_pm2.5_b`, x̄ = (A+B)/2.

**Logic:** High agreement → trust near 100. Large divergence → trust drops toward 0.

**Result:** Mean trust 78–84% across sensors; 290694 slightly lower.

---

### 2. Event Detection (Rule-Based)

**Purpose:** Flag smoke spikes, rain washout, and sensor failures.

**Rules:**

| Event type | Condition | Parameters |
|------------|-----------|------------|
| **Smoke spike** | PM2.5 rise ≥ 50% in 30 min and PM2.5 > 35 µg/m³ | `spike_thresh_pct=0.5`, `spike_window_min=30` |
| **Rain washout** | PM2.5 drop ≥ 35% in 1 h | `washout_thresh_pct=0.35`, `washout_window_min=60` |
| **Sensor failure** | Trust < 40% and A/B divergence > 40% for 3+ consecutive readings | `trust_low_thresh=40`, `ab_divergence_thresh=0.4` |

**Deduplication:** At most one event per type per sensor per 2-hour window.

**Note:** No ground-truth labels; rules are based on domain knowledge.

---

### 3. Forecast with Uncertainty

**Purpose:** Predict next-step PM2.5 and an 80% prediction interval.

**Model:** Random Forest (100 trees, max depth 12).

**Features (no future leakage):**
- 12 lagged PM2.5 values (past 24 min at 2-min resolution)
- Temperature (avg of A/B)
- Humidity (avg of A/B)
- Hour of day
- Day of week

**Target:** PM2.5 at t+1 (next 2 min).

**Train/test split:** Temporal — first 80% train, last 20% test.

**Uncertainty:** 80% interval = 10th and 90th percentile of predictions across the 100 trees (tree bootstrap).

**Results:** MAE 0.51–1.12 µg/m³, R² 0.77–0.98 depending on sensor. Best: 230019 (MAE 0.51, R² 0.98).

## One-Paragraph Pitch (for presentation)

> We built a real-time air-quality monitoring system using a network of low-cost PM2.5 sensors (data from DrKoz/ACS_AQ, Nov 2025). The system (1) assigns a **trust score** to each reading from dual-sensor A/B agreement, (2) detects **events**—smoke spikes, rain washout, sensor failures—via rule-based logic, and (3) **forecasts** short-term PM2.5 with calibrated uncertainty (Random Forest, MAE ~0.5–1.1 µg/m³, R² up to 0.98). The dashboard fetches pre-computed results from our Python pipeline; a Purple Air map is available in the **map** tab for regional context.
